    Question 1

    Coronet Academy school context: classroom IT, database (STUDENT & COURSE tables), grouping in reports, security workshops. (20 marks)

    (a) Identify TWO types of computers that can be used by teachers and students during lessons. (2 marks)

    Any two from: desktop computer, laptop computer, tablet, smartphone, Chromebook / netbook, interactive whiteboard / smart-board PC.

    1 mark each.

    (b) Name ONE hardware device that may be useful for presenting lessons in the classroom. (1 mark)

    Any one of: multimedia projector, interactive whiteboard, smart TV, visualiser / document camera, speakers.

    (c) Identify ONE other storage method by which the school can gain access to its data if the local storage is lost or corrupted. (1 mark)

    Cloud storage / off-site backup (e.g., Google Drive, OneDrive, Dropbox, AWS S3) — copies are stored on remote servers and can be retrieved over the internet if the local copy is lost or corrupted.

    (External hard drive / network-attached storage held off-site is also acceptable.)

    (d)(ii) State the field that would be used as the primary key in the Student table. (1 mark)

    Student ID — it uniquely identifies each student record (no two students share the same Student ID).

    (d)(iii) Identify the field in the Course table that can be used to link the Course table and Student table, and state the name of the key given to that field. (2 marks)

    (d)(iv) The query result shows the course average for each student. State the feature used to generate that course average. (1 mark)

    An aggregate / summary function — specifically the AVG (Average) function applied across the subject-mark fields for each student record. (Also accepted: a "calculated field" in the query, using the AVG function.)

    (d)(v) Explain the purpose of grouping when creating reports in a database. (2 marks)

    Grouping in a database report means arranging records that share a common value (e.g., all students in the same form, all courses in the same department) together in the report output (1 mark).

    This makes the report easier to read and analyse — you can apply summary calculations (totals, averages, counts) within each group, and produce subtotals or sub-headings without having to manually sort the data each time (1 mark).

    (e)(i) Describe TWO ways in which staff can protect their computers and data from possible security threats. (4 marks)
    1. Install and regularly update antivirus & firewall software — antivirus scans for malware (viruses, trojans, ransomware) and removes them; a firewall blocks unauthorised network traffic. Both must be kept up to date so they can recognise the latest threats. (2 marks)
    2. Use strong, unique passwords plus multi-factor authentication (MFA) — passwords should mix upper-case, lower-case, digits and symbols, be changed periodically, and never be shared. MFA (e.g., a code from a phone app) adds a second layer so a stolen password alone is not enough. (2 marks)
    3. Other valid: regular backups; encrypt sensitive files; lock the workstation when stepping away; do not click suspicious links/attachments; install OS & software updates promptly.
    (e)(ii) State TWO personal security measures students should take when they are online. (2 marks)
    1. Do not share personal information (full name, address, phone number, school, photos, location) with strangers online or on public profiles.
    2. Use strong unique passwords for each account and never share them — log out of shared computers.
    3. Other accepted: set social-media profiles to private; do not click on suspicious links or accept friend requests from strangers; tell a parent/teacher if anyone makes them uncomfortable; use a VPN on public Wi-Fi.

    1 mark each.

    Question 2

    Impact Security Services invoice (word-processing); pre-publish website checks; vulnerability vs threat; security threats & phishing. (20 marks)

    (a)(i) State the word-processing feature used to create the dotted line. (1 mark)

    Border / Border style "dotted" applied to a paragraph or row (or, if it sits inside a form line, the underline tab-leader / dotted-line tab leader). Either answer is accepted.

    (a)(ii) Identify the justification that has been applied to all the monetary values. (1 mark)

    Right alignment / Right justification — numeric (currency) values are conventionally right-aligned so the digits line up by place value (units below units, tens below tens, etc.).

    (a)(iii) The company logo is missing from the invoice. The logo is stored in a desktop folder on the sales manager's computer. List, in correct order, the steps that should be taken to insert the logo to the right of the company's name. (4 marks)
    1. Place the cursor (or click) at the position to the right of the company name where the logo should appear.
    2. From the menu, click Insert → Picture (or Image) → From File / This Device.
    3. In the file-browser dialog, navigate to the Desktop folder and select the logo image file (e.g., impact-logo.png); click Insert.
    4. With the picture selected, set Wrap Text → "In line with text" (or use Tight wrap so the image sits beside the company name); resize/reposition with the corner handles so it lines up beside the company name. Save the document.

    1 mark per ordered step.

    (b) Outline TWO checks that should be carried out before publishing a website. (4 marks)
    1. Functionality / link check — click every menu item, button and hyperlink to confirm they all open the correct page; check that forms submit, search works, and downloads complete (no broken links / 404 errors). (2 marks)
    2. Cross-browser & cross-device responsiveness check — view the site in different browsers (Chrome, Safari, Edge, Firefox) and on different screen sizes (desktop, tablet, mobile) to confirm the layout, fonts and images render correctly everywhere. (2 marks)
    3. Other valid checks: spelling and grammar; load-speed test; accessibility (alt text, contrast); SSL / HTTPS certificate; privacy & legal notices in place; SEO meta-tags filled in.
    (c) Define EACH of the following terms in relation to computer security AND provide an example. (i) Vulnerability (2 marks). (ii) Threat (2 marks).

    (i) Vulnerability — a weakness or flaw in a computer system (hardware, software, network or human procedure) that could be exploited by an attacker. (1 mark)
    Example: an outdated operating system that is missing the latest security patches; weak/default passwords; an unencrypted Wi-Fi network. (1 mark)

    (ii) Threat — any potential danger or event that could exploit a vulnerability and cause harm to the system or data. (1 mark)
    Example: a malware/virus attack, a phishing attempt, an insider stealing data, a denial-of-service attack, fire/flood damaging servers. (1 mark)

    (d) State ONE example of EACH of the following security threats and attacks. (4 marks)
    • (i) Industrial espionage — a competing company plants spyware on a rival's network to steal trade secrets / client lists / R&D plans. (Or: a former employee copies confidential design files before leaving and sells them to a competitor.)
    • (ii) Denial-of-service (DoS) attack — flooding a company's website with millions of fake requests so legitimate customers can't access it (e.g., a DDoS botnet attack on an online store on Black Friday).
    • (iii) Ransomware — malware (e.g., WannaCry, LockBit) encrypts all files on the company's servers and demands a Bitcoin payment to release the decryption key.
    • (iv) Data theft — an attacker breaches the customer database and copies/exports millions of customer records (names, emails, credit-card numbers) to sell on the dark web (e.g., the kind of breach reported by major retailers).

    1 mark each.

    (e) State TWO ways in which phishing messages can be sent. (2 marks)
    1. Email (the classic phishing channel — fake emails pretending to be from a bank, courier, social-media site, etc.).
    2. SMS / text message ("smishing") — fake text messages with malicious links.
    3. Other accepted: instant messaging / WhatsApp; voice calls (vishing); social-media direct messages.

    1 mark each.

    Question 3

    Kickstart Investments — networking, intranet/extranet, virus-attack scenario, return-on-investment algorithm. (25 marks)

    (a) Both office locations have their own LAN. Identify the type of network needed to connect both. (1 mark)

    Wide Area Network (WAN) — connects geographically distant LANs (often via the Internet, leased lines or VPN tunnels). A MAN (Metropolitan Area Network) is also accepted if the two offices are within the same city.

    (b) Differentiate between an "intranet" and an "extranet". (2 marks)

    Intranet — a private network inside a single organisation, accessible only by employees / authorised internal users; it uses the same internet technologies (browsers, web pages) but is walled off from the public internet (1 mark).

    Extranet — a portion of the intranet that has been extended to allow access to specific outside parties (customers, suppliers, business partners), usually through secure logins. It sits between the fully private intranet and the fully public internet (1 mark).

    Question 4

    Nigel Sports — verification checks, sports-club spreadsheet, IF function, sorting fields, and a flowchart for a WHILE-loop pseudocode. (25 marks)

    (a)(i) State ONE type of error that could be present after entering data from the application form. (1 mark)

    Transcription error — a typing error such as a transposition (e.g., "1985" entered as "1958"), missed digit, or misspelt name; or a transposition error; or a missing-data error (a required field left blank).

    (a)(ii) Describe TWO verification checks that can be performed when entering data from application forms. (4 marks)
    1. Double-entry verification. The same data is entered twice (often by two different operators) and the system compares the two copies; any mismatch is flagged for the operator to recheck against the source form. (2 marks)
    2. Visual / proofreading verification. The operator (or a second operator) compares the entered record on screen against the original paper application form, line-by-line, to confirm every value matches before saving. (2 marks)

    Note: verification checks confirm the data was entered correctly from the source — they do not check whether the source itself is sensible (that is validation).

    (b)(i) Name the spreadsheet function that will display the number of cells in the range B3:B10 that contain "Tennis". (1 mark)

    =COUNTIF(B3:B10, "Tennis") — counts how many cells in the given range exactly match the criterion "Tennis".

    (b)(ii) Write the formula that was used in Cell E3 to calculate the amount due. (3 marks)

    From the Elite Sports table the cost per session × number of sessions gives the amount due.

    =C3*D3 (cost-per-session × number-of-sessions). If the spreadsheet uses a fixed unit-cost in cell, e.g., =D3*$C$3 (with absolute reference) is also acceptable.

    Mark allocation: 1 mark for using the multiplication operator; 1 mark for the correct cell references; 1 mark for the equals-sign / formula syntax.

    (b)(iii) The IF function performs a logical comparison between two values in a spreadsheet. Describe the syntax of the IF function. (4 marks)

    The IF function takes three arguments:

    =IF( logical_test , value_if_true , value_if_false )

    1. logical_test — a comparison expression that evaluates to TRUE or FALSE (e.g., A1>100).
    2. value_if_true — the value (or formula) returned to the cell if the test is TRUE.
    3. value_if_false — the value (or formula) returned if the test is FALSE.

    Marks: 1 each for the three argument names + 1 for the overall syntax (function name, parentheses and commas).

    Example: =IF(D3>=10, "Discount Member", "Regular") — returns "Discount Member" if D3 is at least 10, otherwise "Regular".

    (b)(iv) Identify both the primary field and the secondary field used to sort the data displayed in the spreadsheet. (2 marks)

    Looking at the order of the rows: rows are first ordered by Activity (Badminton, Basketball, Swimming, Tennis grouped together), then within each Activity rows are alphabetised by Member Name.

    Primary field: Activity. Secondary field: Member Name.

    1 mark each.
