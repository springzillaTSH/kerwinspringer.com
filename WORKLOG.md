## 1. The four-stream offering

For every CSEC subject we cover, the live site eventually delivers **four** things, all free:

| Stream | What it is | Where it lives |
|---|---|---|
| (A) Raw P2 papers | The actual exam PDFs (Paper 2 — structured response) | `papers/{subject}/{january\|june}/` |
| (B) Raw P3 papers | The actual exam PDFs (Paper 3 — SBA / Alternative-to-SBA) | `papers/{subject}/paper3/` |
| (C) P1 MCQ Revision quiz | Interactive practice tool, real questions, instant feedback | `tools/csec-mcq/quiz.html?subject={x}` |
| (D) Worked HTML solutions | Step-by-step P2/P3 solutions, mobile-friendly, with optional PDF export | `tools/p2-solutions/{subject}/{year}/` |

Note: Paper 1 (multiple choice) is **never republished as a PDF + solution**. The legal posture is: practice quiz uses real questions but is reorganized and not labeled by exam-year.

Every AI-generated solution carries the **Kairu attribution**:
> *Solution generated by Kairu — Student Hub's AI system, trained by The Student Hub.*
With the hummingbird mark. When a teacher reviews a solution, the stamp upgrades to "Solution reviewed by Kerwin Springer."

## 5. Procedures (how things are done)

### 5a. Adding a new MCQ paper to an existing subject

```
1. Find PDF in past papers batch:
   03_Teaching Resources/CSEC Past Papers/Fresh new batch to be sorted/CSEC Past Papers/{Category}/{Subject}/Paper 1/{Sitting}/{Year}.pdf

2. Render pages as PNGs:
   pdftoppm -r 150 -png INPUT.pdf /sessions/.../mnt/outputs/{subject}{year}{sitting}/page

3. Copy needed pages (excl. cover) to:
   site-mockup/cropper-pages/{subject}{year}{sitting}/

4. Read each page (Read tool — multimodal). Transcribe each MCQ:
   - Stem (verbatim, but rephrased to drop "Item N refers to" wording)
   - Options A–D in their ORIGINAL positions
   - Correct answer letter (preserve original)
   - Explanation (write yourself)
   - Distractor explanations (one per wrong option)
   - If a diagram is referenced: add a `diagram` field
     — type "image", src "diagrams/{subject}/{q-id}.png"

5. Identify diagram-bearing questions.
   For each, add an entry to the cropper at site-mockup/cropper.html
   with best-guess x/y/w/h. The user opens the cropper, refines, sends JSON back.

6. Once user sends back crop params, run the apply-crops Python script
   (see Section 6) to regenerate all diagram PNGs.

7. Append questions to site-mockup/tools/csec-mcq/data/{subject}.json.
   Update topics list, version, lastUpdated.

8. Update the question count on the MCQ landing page meta line.
```

### 5b. Adding raw P2 / P3 papers

```
1. From the past papers batch, copy PDFs into:
   site-mockup/papers/csec-{subject}/{january|june|paper3}/
   Renamed to: CSEC-{Subject}-Paper-{N}-{Jan|Jun}-{Year}.pdf

2. In papers.html:
   - If new subject: append to subjects[] array (id, name, icon, color, groups)
   - For each paper: append to papers[] array
     {subject, group, year, display, title, file}

3. In index.html (homepage):
   - If new subject: add subject icon to the past papers grid
   - Update FAQ "year coverage" line
   - Update SEO JSON-LD knowsAbout if needed
```

### 5c. Writing an HTML P2 solution

```
1. Use template at: site-mockup/tools/p2-solutions/_template.html
2. Copy to: site-mockup/tools/p2-solutions/{subject}/{year-sitting}/index.html
3. Fill in title, paper metadata (subject, year, sitting, paper count).
4. Read the P2 PDF. For each main question:
   - Question heading (1, 2, 3, 4, 5, 6 ...)
   - Sub-parts (a, b, c with i, ii, iii where applicable)
   - Working shown (KaTeX for math)
   - Final answer in a box / highlight
   - Embed diagrams (SVG redrawn or cropped PNG) as needed
5. Bottom of page: Kairu attribution (or teacher attribution if reviewed)
6. Update papers.html so the year cell links to BOTH the raw PDF AND the
   solution. Show ✅ next to the year label.
```

### 5d. Diagram cropping workflow

```
A) Render page: pdftoppm -r 150 -png paper.pdf /tmp/pages/page
B) Add to cropper.html: a new entry in DIAGRAMS array with id, src,
   stem (description), best-guess x/y/w/h.
C) User opens http://localhost:8000/cropper.html, adjusts sliders,
   clicks "Copy all parameters", pastes JSON back.
D) Apply with Python+PIL: crop the page at user's x/y/w/h, add 8px
   white border, save to tools/csec-mcq/diagrams/{subject}/{id}.png
   (or for P2 solutions, the appropriate path).
E) Reference in JSON / HTML.
```

## 6. Useful one-liners

### Render a PDF to images
```bash
pdftoppm -r 150 -png "input.pdf" /tmp/pages/page
```

### Apply user-supplied crop params (Python)
```python
from PIL import Image, ImageOps
import os, json

CROPS = json.loads('''[USER PASTED JSON]''')
PAGES_BASE = "/sessions/.../mnt/outputs"
OUT_BASE = "/sessions/.../mnt/Claude/site-mockup/tools/csec-mcq/diagrams"

for c in CROPS:
    src = os.path.join(PAGES_BASE, c['src'].replace('cropper-pages/',''))
    img = Image.open(src).convert('L')
    cropped = img.crop((c['x'], c['y'], c['x']+c['w'], c['y']+c['h']))
    bordered = ImageOps.expand(cropped, border=8, fill=255)
    subject = 'biology' if c['id'].startswith('bio-') else 'it'
    out = os.path.join(OUT_BASE, subject, c['id']+'.png')
    bordered.save(out, optimize=True)
```
